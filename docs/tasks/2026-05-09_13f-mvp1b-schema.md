# 13F MVP 1B Schema

## Goal / Acceptance Criteria

Implement execution-plan task `13F-1B-01`: add the MVP 1B filing, parse run, holdings, and temporal CUSIP mapping persistence model.

Acceptance criteria:
- `filings_13f` exposes PRD §7.1 filing fields required by MVP 1B, including coverage, amendment, parser/version, active filing, deadline, totals, and warning/error fields.
- `parse_runs` exists with audit-preserving current-run semantics and independent status enum.
- `holdings_13f` stores holdings through `parse_run_id` with raw value/unit audit fields, normalized `value_usd`, attribution, CUSIP mapping status, and nullable `portfolio_weight_pct`.
- `cusip_ticker_map` uses temporal mapping fields and `mapping_status`, not `is_active` as the lifecycle source.
- Indexes and constraints from PRD §14 are present, including:
  - one active filing per manager/quarter via partial unique index
  - one current parse run per accession via partial unique index
  - unique `(parse_run_id, holding_row_fingerprint)`
  - unique CUSIP mapping candidate key
  - temporal mapping index for `confirmed` / `superseded`
- No parser, SEC fetch, OpenFIGI network, or UI behavior is implemented.

## Scope In

- Alembic migration for schema changes.
- SQLAlchemy model updates.
- Focused unit tests for columns, enums, constraints, and indexes.

## Scope Out

- Filing detail fetch.
- Information table parser.
- Holdings ingestion service.
- OpenFIGI calls or enrichment behavior.
- Admin UI.
- PRD edits.
