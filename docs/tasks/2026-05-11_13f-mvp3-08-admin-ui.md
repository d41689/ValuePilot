# 13F MVP3-08: Admin UI for Historical Backfill, Needs-Validation Queue, and Batch Reparse

## Goal / Acceptance Criteria

Ship the minimal admin surface that makes MVP3's three deferred service contracts
operationally reachable without code access:

1. **Historical Backfill panel** — preview scope, trigger enqueue (with dry_run toggle for
   pre-2023 ranges), and display the `value_unit_risk_warning` / `requires_dry_run` flags
   before the admin commits.
2. **Needs-Validation queue** — list all open
   `QualityFinding13F(rule_code=HISTORICAL_BACKFILL_NEEDS_VALIDATION, status=open)` grouped
   by quarter so ops can see which quarters are waiting for a validation gate to clear.
3. **Batch Reparse panel** — preview by quarter (quarter scope ships first per D3), enqueue
   with the deduplication lock warning, show per-status filing counts from the impact summary.
4. **Corporate Action confirmation UI** — a dialog/drawer over the existing CUSIP mapping
   table that calls the already-live `/cusips/corporate-actions/{preview,confirm}` endpoints;
   shows affected ownership_changes count before the admin submits evidence + reason.

Acceptance criteria:
- All four surfaces use the existing admin auth gate (`AdminUser` dependency / admin session).
- Historical backfill preview shows `value_unit_risk_warning`, `requires_dry_run`, and the
  per-manager-quarter scope count before the enqueue button is enabled.
- Enqueue button is disabled while a job with the same `lock_key` is active
  (`queued` / `running` / `cancel_requested`).
- Corporate action confirm dialog requires non-empty `evidence_url` and `reason` before the
  submit button is enabled (mirrors service-level D4 guard).
- Batch reparse panel exposes quarter scope only; manager scope input is present but gated
  behind a "Coming soon" note per D3 UI-rollout constraint.
- Needs-Validation queue shows open finding count per quarter with a link to the quality
  report that generated them.
- Backend: three new endpoint groups added to `thirteenf_admin.py`; existing service modules
  are not modified.
- All new backend endpoint unit tests pass in Docker.
- Frontend: no regressions in existing admin panel behavior.

## Scope In

**Backend endpoints (new routes in `thirteenf_admin.py`):**
- `POST /admin/13f/backfill/preview` → `preview_historical_backfill`
- `POST /admin/13f/backfill/enqueue` → `enqueue_historical_backfill`
- `GET  /admin/13f/backfill/needs-validation` → query open
  `HISTORICAL_BACKFILL_NEEDS_VALIDATION` findings grouped by quarter
- `POST /admin/13f/jobs/reparse-by-quarter/preview` → `preview_batch_reparse(quarter=...)`
- `POST /admin/13f/jobs/reparse-by-quarter/enqueue` → `enqueue_batch_reparse(quarter=...)`

**Frontend UI additions to `app/(dashboard)/admin/13f/page.tsx`:**
- Historical Backfill card panel (start quarter, end quarter, dry_run toggle, preview call,
  risk warning banner, enqueue button)
- Needs-Validation queue card (per-quarter table, open count, quality_report link)
- Batch Reparse card panel (quarter input, preview call, filing count estimate, enqueue
  button; manager scope input rendered but disabled with "D3: UI ships quarter scope first")
- Corporate Action confirmation drawer/dialog (CUSIP input, lookup existing mappings, preview
  affected count, evidence_url / reason fields, effective_from_quarter, submit → confirm)

**Tests:**
- `backend/tests/unit/test_13f_mvp3_admin_ui_endpoints.py` — new endpoint contract tests
  (auth gate, 400 on service errors, response shape for each new route)
- Frontend: no new automated frontend tests beyond type-check pass (existing pattern for
  this admin page)
